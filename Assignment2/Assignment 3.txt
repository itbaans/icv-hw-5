Project Context
Your Assignment 1 classical image processing pipeline has been deployed and tested. The client is expanding operations and now has TWO processing lines with different imaging systems:
Dataset 1 (Seeds): Original seed processing line with clustering-based approach
Dataset 2 (BSDS500 - Natural Images): Berkeley Segmentation Dataset with complex natural scenes requiring contour detection
Both datasets struggle with:
* Clear boundary detection between touching/overlapping objects
* Varying edge sharpness due to lighting conditions
* Objects with different sizes and complex textures
Your team needs to develop Version 2.0 using edge detection methods (Canny and Sobel) to improve boundary detection across BOTH datasets. The BSDS500 dataset will test whether your edge detection approach generalizes to complex natural scenes beyond agricultural products.
	

________________


Assignment Objectives
By completing this assignment, you will:
* Apply edge detection methods (Canny, Sobel) on two different datasets
* Compare edge detection performance across different object types 
* Analyze which features work best for which dataset
* Compare feature-based methods against your classical baseline


Datasets
Dataset 1: Seeds (from Assignment 2)
You will use the same seed dataset from Assignment 1 with your intermediate_outputs/ folder containing:
* Preprocessed filtered images
* Ground truth seed counts
* Clustering results from Assignment 2 as baseline


Dataset 2: BSDS500 (Berkeley Segmentation Dataset)
📊 BSDS500 Dataset Information
Source: University of California, Berkeley (2001)
Download: https://beta.hyper.ai/en/datasets/5379
Description: 500 natural images with human-annotated boundaries and segmentations. Contains 200 training images and 100 test images.
Use for this assignment: Select 50-100 images from the training set for edge detection experiments
Ground Truth: Human-annotated boundary maps available for comparison (optional for analysis)
Why BSDS500? Standard benchmark for contour/edge detection algorithms. Tests whether your edge detection pipeline generalizes from simple objects (seeds) to complex natural scenes.
